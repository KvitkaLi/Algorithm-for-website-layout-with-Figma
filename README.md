## Алгоритм верстки проєкту вебсторінки
Алгоритм верстки вебсторінки — це послідовний процес перетворення графічного макету (наприклад, із Figma) у робочий код за допомогою HTML, CSS та JavaScript.  
Основні етапи включають аналіз макету, створення структури сторінки, стилізацію, адаптацію під різні пристрої та тестування. Це забезпечує коректне відображення сайту у різних браузерах і на різних екранах.  
________________________________________  
### Основні етапи верстки вебсторінки
### 1. Аналіз макета у Figma  
Перед початком верстки необхідно уважно вивчити макет.  
•	Визначте загальну структуру сторінки (шапка, основний контент, підвал).  
•	Проаналізуйте блоки сторінки та їхню ієрархію.  
•	Визначте спільні стилі: кольори, шрифти, відступи (spacing).  
•	Перевірте адаптивні версії макету (desktop, tablet, mobile).  
•	Підготуйте графічні елементи:  
o	експортуйте зображення  
o	експортуйте іконки (бажано у форматі SVG).  
________________________________________  
### 2. Підготовка структури проєкту  
Перед початком верстки створюється структура папок.  
Приклад структури проєкту:  
<img width="769" height="513" alt="image" src="https://github.com/user-attachments/assets/c6519db1-b8bb-4623-9105-75fb22c06bc2" />

Рекомендації:  
•	використовуйте зрозумілі назви файлів  
•	назви пишіть латиницею  
•	не використовуйте пробіли у назвах.  
________________________________________  
### 3. Створення HTML-структури (скелету сторінки)  
Після підготовки структури створюється HTML-код, який визначає логічну будову сторінки.  
Базові налаштування HTML:  
**У <head> необхідно додати:** 
<meta charset="UTF-8">  
<meta name="viewport" content="width=device-width, initial-scale=1.0">  
<title>Назва сторінки</title>.    
**Особливо важливий: viewport - без нього адаптив не працює правильно.**
###  Семантична структура  
Для структурування сторінки використовують семантичні теги:  
* <header> — шапка сайту  
* <nav> — навігація  
•	<main> — основний контент  
•	<section> — розділи сторінки  
•	<article> — окремий матеріал  
•	<footer> — підвал сайту  
________________________________________  
### 4. Підключення стилів і шрифтів  
Після створення HTML потрібно підключити стилі та шрифти.  
•	підключіть файл стилів style.css  
•	підключіть шрифти (наприклад Google Fonts або локальні).  
________________________________________  
### 5. Налаштування базових стилів  
На цьому етапі виконуються глобальні налаштування стилів.  
Скидання стилів браузера.  
Підключається Reset або Normalize CSS.  
Базові стилі:  
Налаштовуються:  
•	шрифти  
•	кольори  
•	базові відступи  
•	розміри тексту.  
Приклад використання змінних CSS:  
:root{  
    --primary-color:#222F33;  
    --font-size:14px;  
}  
________________________________________    
### 6. Створення основного контейнера сторінки.    
Для зручної верстки сторінка часто має основний контейнер.  
Приклад:  
.wrapper {  
   min-height: 100%;  
   display: flex;  
   flex-direction: column;  
}   
.page {  
   flex: 1 1 auto;  
}  
Це дозволяє правильно розташувати елементи сторінки та зафіксувати footer внизу сторінки.  
________________________________________  
### 7. Верстка блоків сторінки  
Верстка виконується відповідно до макету.  
Для розташування елементів використовуються сучасні інструменти CSS:  
•	Flexbox  
•	Grid  
Під час верстки необхідно:  
•	дотримуватися структури макету  
•	правильно виставляти відступи  
•	контролювати розміри елементів.  
________________________________________  
### 8. Налаштування контейнера сторінки  
Часто використовується обмежуючий контейнер.  
**Приклад:**  
[class*="__container"] {  
	max-width: 1200px;  
	padding-left: 0,9375rem; /* відповідно 15px*/;   
	padding-right: 0,9375rem;  
	margin: 0 auto;  
}  
Це дозволяє центрувати контент і контролювати ширину сторінки.  
________________________________________  
### 9. Адаптивна верстка (Responsive Design)  
Сторінка повинна коректно відображатися на різних пристроях.  
Для цього використовуються медіазапити.  
@media (max-width: 768px) {  
}  
Потрібно перевірити вигляд сторінки на:  
•	desktop  
•	tablet  
•	mobile.  
________________________________________  
### 10. Оптимізація зображень  
Щоб сторінка завантажувалась швидше:  
•	використовуйте формат SVG для іконок  
•	використовуйте WebP для фотографій  
•	задавайте:  
max-width: 100%;  
height: auto;  
________________________________________  
### 11. Тестування та налагодження  
Після завершення верстки необхідно перевірити роботу сторінки.  
**Основні етапи тестування:**    
•	перевірка у різних браузерах    
•	перевірка адаптивності  
•	перевірка відображення шрифтів  
•	перевірка роботи інтерактивних елементів.  
Також бажано перевірити код у валідаторі HTML та CSS.  
________________________________________  
### Основні поширені помилки  
•	Некоректне використання CSS — може призвести до неправильного відображення елементів.  
•	Відсутність адаптивності — сайт погано працює на мобільних пристроях.  
•	Використання занадто великих зображень — уповільнює завантаження сторінки.  
•	Помилки в HTML-коді — неправильне закриття тегів або помилки у вкладеності.  
•	Ігнорування семантичних тегів — погіршує доступність та SEO сторінки.

### Приклад простої схеми   
Ось проста схема структури вебсторінки, яку зазвичай використовують під час верстки. Така схема добре допомагає зрозуміти ієрархію HTML-елементів перед початком роботи.  
Схема структури вебсторінки (HTML)  
wrapper  
│  
├── header  
│     ├── logo  
│     └── nav  
│  
├── main  
│     │  
│     ├── hero / banner  
│     │  
│     ├── section  
│     │     ├── title  
│     │     └── content  
│     │  
│     ├── section  
│     │     ├── cards  
│     │     │     ├── card  
│     │     │     ├── card  
│     │     │     └── card  
│     │  
│     └── section  
│           └── gallery / features / article  
│  
└── footer 
      ├── contacts  
      ├── navigation  
      └── copyright

### Приклад відповідної HTML-структури  
<div class="wrapper">  

<header class="header">  
   <div class="header__container">  
      <div class="logo"></div>  
      <nav class="menu"></nav>  
   </div>  
</header>  

<main class="page">  

   <section class="hero"></section>  

   <section class="about">  
      <div class="about__container"></div>  
   </section>  

   <section class="cards">  
      <div class="cards__container">  
         <div class="card"></div>  
         <div class="card"></div>  
         <div class="card"></div>  
      </div>  
   </section> 

</main>  

<footer class="footer">  
   <div class="footer__container"></div>  
</footer>  

</div> 
Header → верх сайту (логотип, меню).  
Main → основний контент.  
Footer → підвал (контакти, копірайт).
